# Launch Checklist

## Цель

Этот чек-лист нужен для финальной ручной проверки личного сайта веб-разработчика перед production launch и запуском рекламы.

Использование:
- проходить вручную перед релизом
- отмечать только реально проверенные пункты
- если не закрыт любой пункт из блока Critical Before Launch, сайт не считается готовым к запуску рекламы

---

# 1. Full Launch Checklist

## 1.1 Content

### Core messaging
- [ ] За 3–5 секунд понятно, кто я, что я делаю и для кого
- [ ] Hero section содержит ясный оффер, а не только красивую визуальную подачу
- [ ] Подзаголовок объясняет ценность услуги
- [ ] Primary CTA понятен и логично связан с оффером
- [ ] Текст сайта звучит уверенно, современно и профессионально без воды и фальши
- [ ] На сайте нет lorem ipsum, временных заглушек и placeholder-контента
- [ ] Нет блоков с текстами типа “coming soon”, “TBD”, “replace later”

### Services and trust
- [ ] Услуги описаны понятно и конкретно
- [ ] Есть объяснение, что именно входит в работу
- [ ] Есть причина доверять: кейсы, подход, опыт, процесс, стек, личная подача
- [ ] Блок “обо мне” усиливает доверие, а не просто заполняет место
- [ ] Кейсы показывают не только картинку, но и смысл / результат / задачу
- [ ] Контакты указаны корректно и актуально
- [ ] FAQ, если есть, действительно снимает возражения

### Consistency and quality
- [ ] На всех страницах единый tone of voice
- [ ] CTA формулировки согласованы между собой
- [ ] Нет орфографических и пунктуационных ошибок
- [ ] Нет визуально слишком длинных абзацев без структуры
- [ ] Все названия секций, услуг и кнопок финальные
- [ ] Контент не выглядит шаблонным или собранным “по клише”

### Nice to have later
- [ ] Расширенные кейсы с процессом и результатом
- [ ] Блог / заметки / insights
- [ ] Дополнительные страницы под SEO и отдельные услуги

## 1.2 UI / UX

### Structure and clarity
- [ ] Визуальная иерархия понятна: headline → explanation → CTA
- [ ] На каждой ключевой странице есть один явный primary CTA
- [ ] Длинные страницы имеют повтор CTA в логичных местах
- [ ] Пользователь понимает следующий шаг на каждой странице
- [ ] Навигация не перегружена
- [ ] Header понятный и не отвлекает от конверсии
- [ ] Footer содержит нужные переходы и контакты

### Visual quality
- [ ] Сайт выглядит цельно: типографика, отступы, радиусы, карточки, кнопки, сетка
- [ ] Нет визуально “ломаных” секций
- [ ] Нет перегруженных блоков, где много всего одновременно
- [ ] Эффекты, видео и motion усиливают восприятие, а не мешают
- [ ] Нет элементов, которые выглядят кликабельными, но не кликаются
- [ ] Hover / active / focus states настроены
- [ ] Ошибки, success-state, empty-state оформлены аккуратно

### Interaction quality
- [ ] Меню открывается и закрывается предсказуемо
- [ ] Модалки / drawers / overlays не ломают UX
- [ ] Sticky элементы не перекрывают важный контент
- [ ] Скролл ощущается плавно и стабильно
- [ ] Нет неожиданных скачков layout при загрузке
- [ ] Нет визуальных багов при reload / back / forward navigation

### Nice to have later
- [ ] Дополнительные микроинтеракции
- [ ] Более сложный storytelling motion
- [ ] A/B варианты hero / CTA / структуры

## 1.3 Responsive

### Core breakpoints
- [ ] Проверен mobile portrait
- [ ] Проверен mobile large
- [ ] Проверен tablet
- [ ] Проверен laptop
- [ ] Проверен desktop
- [ ] Проверен wide desktop

### Layout behavior
- [ ] Нет горизонтального скролла
- [ ] Нет обрезанных заголовков
- [ ] Нет обрезанных кнопок
- [ ] Нет наезжающих друг на друга блоков
- [ ] Изображения и видео не ломают layout
- [ ] Карточки и сетки корректно перестраиваются
- [ ] Декоративные элементы не перекрывают текст
- [ ] Sticky/fixed UI не мешает контенту

### Mobile UX
- [ ] На мобильном легко нажимать на кнопки и ссылки
- [ ] Меню удобно использовать одной рукой
- [ ] Формы удобно заполнять на мобильном
- [ ] Hero не разваливается на маленьких экранах
- [ ] Текст читается без зума
- [ ] CTA не уезжают слишком низко
- [ ] Анимации не мешают мобильному UX

### Edge cases
- [ ] Проверены длинные имена и длинные email
- [ ] Проверены длинные многострочные заголовки
- [ ] Проверены случаи с отсутствием медиа / fallback
- [ ] Проверены промежуточные брейкпоинты, а не только типовые

### Nice to have later
- [ ] Отдельно оптимизированные mobile motion patterns
- [ ] Специальные упрощённые mobile-версии тяжелых секций

## 1.4 Forms / Leads

### Basic functionality
- [ ] Все формы реально отправляют данные
- [ ] Данные доходят туда, куда должны: email / CRM / таблица / backend / webhook
- [ ] Success-state показывается после успешной отправки
- [ ] Error-state показывается при неуспешной отправке
- [ ] Кнопка submit имеет loading-state
- [ ] Повторный клик по submit не создаёт дубли

### Validation
- [ ] Обязательные поля валидируются
- [ ] Email валидируется корректно
- [ ] Телефон, если есть, валидируется адекватно
- [ ] Очень длинные значения не ломают форму
- [ ] Спецсимволы не ломают форму
- [ ] Пустые поля обрабатываются корректно
- [ ] Ошибки понятны пользователю

### Reliability
- [ ] Форма протестирована при медленном интернете
- [ ] Форма не ломается, если backend отвечает с задержкой
- [ ] Форма не отправляется дважды при reload / back / repeated click
- [ ] Все CTA, ведущие к форме, работают корректно
- [ ] Thank-you page, если есть, открывается корректно
- [ ] Thank-you page не индексируется, если не должна

### Anti-spam and attribution
- [ ] Есть защита от спама: honeypot / captcha / rate limiting / server validation
- [ ] Источник лида сохраняется
- [ ] UTM-метки доходят до лида или хотя бы сохраняются в аналитике
- [ ] Отправка формы отслеживается как conversion event

### Manual lead path
- [ ] Пройден полный путь вручную:
  - [ ] открыть страницу
  - [ ] нажать CTA
  - [ ] заполнить форму
  - [ ] отправить
  - [ ] увидеть success-state
  - [ ] убедиться, что лид дошёл
  - [ ] убедиться, что аналитика записала событие

### Nice to have later
- [ ] Многошаговый бриф
- [ ] Автоответ пользователю
- [ ] Квалификация лида на уровне формы
- [ ] Автоматическая маршрутизация заявок

## 1.5 SEO

- [ ] У каждой ключевой страницы есть уникальный title
- [ ] У каждой ключевой страницы есть уникальный meta description
- [ ] У каждой ключевой страницы корректный H1
- [ ] На странице только один H1
- [ ] Заголовочная иерархия H2/H3/H4 логична

### Indexation and crawl
- [ ] robots.txt существует и настроен корректно
- [ ] sitemap.xml существует и актуален
- [ ] Нет случайной блокировки важных страниц
- [ ] Тестовые, технические и thank-you страницы исключены из индекса при необходимости
- [ ] Canonical указан там, где нужен
- [ ] Нет дублей title / description на ключевых страницах

### Page quality
- [ ] URL чистые и читаемые
- [ ] Нет временных slug
- [ ] Есть базовая внутренняя перелинковка
- [ ] Alt-тексты добавлены там, где изображения несут смысл
- [ ] Open Graph и social preview настроены
- [ ] Favicons и social preview изображения отображаются корректно

### Nice to have later
- [ ] Schema markup
- [ ] Отдельные SEO-страницы под услуги

- [ ] Аналитика установлена на production
- [ ] Pageview записываются корректно
- [ ] Нет дублей pageview
- [ ] Ключевые conversion events настроены
- [ ] Названия событий единообразны и понятны

### Conversion tracking
- [ ] Отправка формы фиксируется как conversion
- [ ] Клик по primary CTA фиксируется
- [ ] Клик по email фиксируется
- [ ] Клик по телефону / мессенджеру фиксируется
- [ ] Переход на thank-you page, если есть, фиксируется
- [ ] Конверсии проверены вручную через debug / preview / realtime

### Ads readiness
- [ ] Подключены нужные pixels / tags для рекламных систем
- [ ] Проверено, что пиксели реально срабатывают
- [ ] UTM-метки сохраняются
- [ ] Источник кампании можно отследить
- [ ] Нет конфликтов аналитики с consent logic

### Data quality
- [ ] Внутренний трафик исключён по возможности
- [ ] Нет лишних тестовых событий
- [ ] Нет дублирующих триггеров на одно действие

### Nice to have later

- [ ] Scroll tracking
- [ ] Section visibility tracking
- [ ] Heatmaps / session replay

## 1.7 Performance

### Hero and first load
- [ ] Первый экран быстро появляется
- [ ] Hero не перегружен тяжелыми ресурсами
- [ ] Above-the-fold контент загружается быстро
- [ ] Нет явных layout shifts в hero
- [ ] Ключевой CTA доступен сразу

### Media optimization
- [ ] Изображения оптимизированы по размеру
- [ ] Используются современные форматы изображений там, где уместно
- [ ] Видео не грузятся агрессивно без необходимости
- [ ] Lazy loading используется там, где это уместно
- [ ] Есть fallback для тяжелого медиа

### Frontend performance
- [ ] Bundle size не раздут лишними библиотеками
- [ ] Нет тяжелых зависимостей ради мелкого визуального эффекта
- [ ] Анимации не вызывают лагов
- [ ] Scroll-based эффекты работают плавно
- [ ] На low-end devices сайт остаётся usable
- [ ] Нет ощутимого input lag
- [ ] Шрифты загружаются аккуратно и не блокируют рендер

### Testing conditions
- [ ] Проверено на реальном мобильном устройстве
- [ ] Проверено на слабом интернете
- [ ] Проверено с включённым reduced motion
- [ ] Проверено поведение без autoplay-heavy media

### Nice to have later
- [ ] Дополнительный audit bundle
- [ ] Тонкая оптимизация animation pipeline
- [ ] Отдельная low-power стратегия для тяжелых секций

## 1.8 Accessibility Basics

### Keyboard and focus
- [ ] Все ключевые интерактивные элементы доступны с клавиатуры
- [ ] Focus states видимы
- [ ] Порядок фокуса логичен
- [ ] Модалки и меню не ломают фокус

### Semantics and readability
- [ ] Используется семантическая HTML-структура
- [ ] Текст имеет достаточный контраст
- [ ] Текст читается на мобильном и desktop
- [ ] Кнопки и ссылки различимы не только цветом
- [ ] Формы имеют label или доступный эквивалент
- [ ] Сообщения об ошибках формы понятны и связаны с полем

### Motion and media
- [ ] Есть базовая поддержка prefers-reduced-motion
- [ ] Автовоспроизводящееся видео не мешает использованию сайта
- [ ] Декоративные изображения не засоряют accessibility tree
- [ ] Смысловые изображения имеют alt

### Nice to have later
- [ ] Screen reader walkthrough
- [ ] Более глубокий a11y audit
- [ ] Улучшение aria-атрибутов в сложных интерактивах

## 1.9 Technical Setup

### Domain and security
- [ ] Production domain подключён
- [ ] HTTPS работает
- [ ] Редирект HTTP → HTTPS настроен
- [ ] Выбрана и корректно настроена каноническая версия домена
- [ ] SSL-сертификат валиден

### Environment and integrations
- [ ] Все production env variables заданы
- [ ] Нет утечки секретов в client bundle
- [ ] API routes работают в production
- [ ] Webhooks работают в production
- [ ] Email sending / notifications работают в production
- [ ] Подключены production credentials, а не test credentials

### Stability and cleanup
- [ ] Нет критичных ошибок в browser console
- [ ] Нет критичных network errors
- [ ] Проверены внутренние ссылки
- [ ] Проверены внешние ссылки
- [ ] 404 страница настроена
- [ ] 500 / error boundary поведение продумано
- [ ] Удалены тестовые страницы и debug-элементы
- [ ] Удалены ненужные console.log и debug-инструменты
- [ ] Favicon и app icons подключены

### Legal / operational basics
- [ ] Privacy policy добавлена, если нужна по стеку и аналитике
- [ ] Consent / cookie logic работает корректно, если используется
- [ ] Есть базовый backup критичных конфигов и контента
- [ ] Есть понятный rollback plan

### Stability / observability
- [ ] Error monitoring подключён
- [ ] Есть базовый способ быстро увидеть 404 / 500 / ошибки форм

### Nice to have later
- [ ] Uptime monitoring
- [ ] Alerting в Telegram / Slack / email
- [ ] Формализованный release flow

# 3. Post-launch checks

## 3.1 Сразу после релиза
- [ ] Открыть production URL в приватном окне
- [ ] Пройти homepage как новый пользователь
- [ ] Нажать primary CTA
- [ ] Отправить тестовую заявку
- [ ] Проверить, что заявка дошла
- [ ] Проверить, что конверсия записалась
- [ ] Проверить мобильную версию на реальном устройстве
- [ ] Проверить консоль и сеть на критичные ошибки
- [ ] Проверить social preview при шаринге ссылки

## 3.2 В первые 24 часа
- [ ] Проверить realtime / live analytics
- [ ] Убедиться, что pageview и события идут корректно
- [ ] Проверить, что рекламный трафик приходит на нужные страницы
- [ ] Проверить, что источники и UTM фиксируются
- [ ] Проверить, что нет аномалий по формам
- [ ] Проверить, что нет всплеска 404 / 500 ошибок
- [ ] Проверить скорость сайта на мобильном интернете

# 5. Отдельный блок для запуска рекламы

## Landing-message match
- [ ] Сообщение на рекламном креативе совпадает с hero / above-the-fold сайта
- [ ] Пользователь после клика сразу понимает, куда попал
- [ ] Страница не перегружена лишними отвлекающими блоками до первого CTA

## Conversion readiness
- [ ] На первом экране или сразу после него есть понятное действие
- [ ] Контакты и форма работают без сюрпризов
- [ ] Есть хотя бы один сильный блок доверия до формы
- [ ] Весь путь от входа до заявки укладывается в понятную логику
