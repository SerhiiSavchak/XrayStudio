Page Structure Spec — Personal Web Developer Website
1. Goal

Спроектировать структуру ключевых страниц личного сайта веб-разработчика так, чтобы каждая страница имела ясную роль в воронке, усиливала доверие, поддерживала premium digital product feel и вела пользователя к заявке без перегруза контентом и визуалом.

2. Context

Сайт должен работать как:

коммерческий инструмент привлечения клиентов

демонстрация уровня как разработчика

premium digital product experience

масштабируемая основа для дальнейшего роста проекта

Структура страниц должна быть:

понятной

конверсионной

визуально сильной

SEO-aware

performance-aware

3. Requirements

Для каждой ключевой страницы необходимо определить:

цель страницы

основное пользовательское понимание

целевое действие

обязательные блоки

роль страницы в UX, trust, conversion, SEO

Ключевые страницы:

Homepage

Services

Cases / Selected Work

About

Contact

FAQ

4. Definition of Done

Структура страниц считается готовой, если:

у каждой страницы есть одна основная роль

нет дублирования функций между страницами

каждая страница поддерживает следующий шаг пользователя

контент можно масштабировать без переделки логики

MVP-версия может быть реализована без перегруза

7. Risks and Constraints

Основные риски:

перегруженная homepage

кейсы без narrative

размытая services page

слишком длинная about page

слишком сложная contact form

визуальный перегруз ради wow-effect

SEO в ущерб ясности

8. Final recommendation

Оптимальная структура для старта:

сделать Homepage как мощный, но не перегруженный хаб

сделать Services как страницу коммерческой ясности

сделать Cases как curated portfolio, а не архив

сделать About компактной, но сильной

сделать Contact максимально лёгкой

сделать FAQ короткой и прикладной

Итоговая логика сайта:
Homepage → Services / Cases → About / FAQ → Contact
